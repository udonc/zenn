2026年になってしまいましたが、State of chot Inc. 2025 の集計結果を発表します！

## State of chot Inc. とは？

世界中のJavaScriptエンジニアのアンケートをもとに業界の動向やトレンドを調査する State of JavaScript というものがあります。
それにならって、ちょっと社で働く皆さんのあれこれについて統計を取ってみようという企画です！
（名前に "2025" と入っていますが、毎年やるかは未定です…… ！）

### 調査方法

- Google フォームを用いて回答を募集
- 回答期間: 12/15 ~ 12/22
- 集計元データ: Google Formの回答

## 🌐 Webブラウザ (デスクトップ)

![デスクトップブラウザの利用状況](/images/state-of-chot-inc-2025/browser-usage-detailed.png)

利用者トップはChromeで、Safari, Edge, Arcが後に続く結果になりました！

### 使ってる理由・気に入っているところ（抜粋）

#### Chrome
制作物の表示確認用 - エンジニア
Googleのアプリをよく使うから - マーケティング
なんとなくです！ - コーディネーター

#### Safari

使い慣れてるから。あとはサイトの見え方確認とかで使ってます - マーケティング

#### Edge

縦タブ - エンジニア

#### Arc

プロファイル切り替えの容易さ、Pinned Tabsの視認性・操作性の良さ、ページプレビュー機能の便利 - エンジニア
SidebarファーストなUI設計とSpaceとCommand Barが馴染みすぎて離れられない… - エンジニア

#### Firefox

Arcが開発終了したタイミングで、縦タブがネイティブサポートされているのはFirefoxだけだったので乗り換えました。モバイルとの連携が強く、タブ移動での動画ポップアップにも対応しているのも地味に嬉しい。 - エンジニア

#### Brave

全体的に使いやすい感じがする。タブの縦並びは必須要件。 - エンジニア

#### Vivaldi

タブが左側に配置できたり、カスタマイズがとにかく豊富で、自分好みのツールを使いたいこだわりが満たせる。 - エンジニア

#### Dia

英語のドキュメントを読む時に翻訳や要約してもらったりする。日本語のドキュメントでも言葉が難しい時は優しい言葉にしてとお願いすると読めるようになるので便利。 - エンジニア

#### Zen

Arc からの移行でプライベートで使ってます。アドレスバーとかを非表示にしてほぼ全画面みたいにできるところが好き - エンジニア

#### LibreWolf

プライバシー保護 - エンジニア

### AIツールの活用法（抜粋）

- 退勤前、Claude Code に明日やることを話して「もし忘れてたら注意して！」って言っておくと明日の自分が嬉しい という使い方を最近覚えました
- コードだけじゃなくて設計壁打ちとか思考実験に付き合ってもらえるので、コードとかIDE以外にもいろんな視点を持つと良いと思います。
- 資料のアイディア出し、壁打ち、スプシの表＆計算、情報収集など
- エディタでの自動化が難しい繰り返しの単純作業、自分で中々気付けないバグ原因を聞く、設計検討など

## ✅ タスク管理ツール

Notionがトップですが、使っていない方やシンプルなメモのみで運用している方も多そうでした！

### 使ってる理由・気に入っているところ（抜粋）

#### Notion

NotionはDB管理ができるので、タスク管理と案件を紐づけて、タスク単位での管理ではなく、案件単位でのタスク管理ができるのが気に入っています。 - ディレクター

#### Todoist

自然言語入力でタスクを作成できるから ("10日ごと" や "毎月15日" と入力するとその通りに日時設定してくれる) 。 - エンジニア

#### Google Tasks

Googleカレンダーからアクセスできるのが嬉しいです - エンジニア

#### Obsidian

マークダウンの使い勝手やフォルダで管理するところがいいのでメモ系ツールはObsidianに統一した。プラグインを入れて自分好みにカスタマイズできるところが気に入っている。見た目(カラーなど)やマークダウンで入力したときのスタイルも変更できるのでよい。 - エンジニア

## 📝 エディタ

![エディタの利用状況](/images/state-of-chot-inc-2025/editor-usage.png)

Visual Studio Codeが圧倒的にトップで、Cursor, Neovim, Zed が後に続くような結果になりました！

### 使ってる理由・気に入っているところ（抜粋）

#### Visual Studio Code

- 一番標準的に使えるので
- 特にこだわりがないため
- デファクトで拡張機能のサポートとかも手厚いから
- Devcontainerが好きだから

#### Cursor

- 会社のAI課金で対象だったのと、かっこいいので
- 最近アップデートが多くて楽しい
- AI化が進んだ時に触り始めてVSCodeとの差分がないから使いやすい

#### Neovim

- 見た目がかっこいいので……これは Vim でもそうですが、定期的にTwitterとかで「知らなかった便利コマンド」に出会えるのが楽しいです。一生使いこなせないところが好き。
- VSCodeには必要の無い機能が多すぎると感じているので

#### Helix

- 軽い シンプル lsp対応

#### Vim

- サーバ管理など GUI の無い環境でもいつも通り使えるところ、ターミナルでコマンド操作中心の習慣と相性が良いところ

## 💻 ターミナル

![ターミナルの利用状況](/images/state-of-chot-inc-2025/terminal-usage.png)

エディタ組み込みのものやmacOS標準のターミナルを使っている方が多く、サードパーティーとしてはiTerm2やWarpが支持されている結果となりました！

### 使ってる理由・気に入っているところ（抜粋）

#### エディタ組み込みのもの

- エディタからのアクセスが最速なので
- ディレクトリ移動しなくて済むのがとても好き。些細なことですが

#### macOS標準ターミナル

- code コマンドさえ打てたらいいので。
- 機能的にデフォルトのもので満足

#### iTerm2

- 昔から使ってて十分だと思っています
- 最近はCLIでの複雑な操作をしなくなったのもあり、あまりこだわりがなく惰性で使っている

#### Warp

- プラグインを入れずとも最初から整っている。入力補完、検索、クリックでカーソル移動できたり範囲選択できるところ。
- 補完がよく効くし、VSCodeみたいにFinderからディレクトリを開けるのがいい。あとちょっとした編集ならWarpのGUIで対応できるのが嬉しい。

#### Ghostty

- 出たときに試しに移行してから、特に不満もないのでそのまま使っている感じです。正直 WezTerm の方が Lua でカスタマイズできるので好きかもです。
- tmuxを使わずともpain分割でき、macOS標準ターミナルをラップしてグラフィックのところパフォーマンス改善してくれている（？）らしいので使ってます。アイコンも可愛い👻
